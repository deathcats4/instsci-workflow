# Contributing

This preview is most useful when feedback is concrete and reproducible.

## Good Issue Reports

Please include:

- Operating system and Python version.
- InstSci command used.
- Publisher or DOI prefix involved.
- Relevant manifest row, with private paths/tokens removed.
- Whether the result was OA, publisher-open PDF, browser-verified, or not verified.
- What you expected and what happened instead.

Do not include passwords, cookies, SSO screenshots with personal data, full browser profiles, or subscription-only PDFs.

## Useful Test Cases

Good public-beta test batches include:

- OA direct success.
- Publisher open PDF success.
- Institution login required.
- Access unavailable through your institution.
- WAF or human verification.
- Publisher error.
- PDF candidate conflict.
- Zotero item plus linked PDF sync.

## Project Boundaries

This project is for lawful, diagnosable literature acquisition workflows. It should not add credential automation, CAPTCHA bypass, publisher access-control bypass, or hidden-browser SSO behavior.
